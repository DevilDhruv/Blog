    Merge Two Sorted Arrays in JavaScript

    This problem looks deceptively simple, yet it reveals how well you think about efficiency, constraints, and iteration. In this blog, we‚Äôll walk through the exact thought process an interviewer expects ‚Äî from the naive approach to the optimal one.

    üìå Problem Statement

    You are given two arrays that are already sorted in ascending order. Your task is to merge them into a single sorted array. 

    
const a = [2, 4, 7, 9];
const b = [1, 3, 6, 8];

// Expected Output:
// [1, 2, 3, 4, 6, 7, 8, 9]

    The constraint that both arrays are already sorted is the most important clue in this problem ‚Äî and ignoring it usually leads to suboptimal solutions.

    üß† How an Interviewer Expects You to Think

    A common beginner instinct is:

    • Combine both arrays
    • Sort the result

    While this works, it immediately raises a red flag in interviews. Sorting again costs O((n + m) log(n + m)), which is unnecessary.

    Since both arrays are already sorted, the interviewer expects you to ask:

    ‚ÄúCan I merge them in one pass without re-sorting?‚Äù

    This leads us to the two pointer technique.

    üõ† Step-by-Step Thought Process

    We maintain one pointer for each array:

    • aIndex ‚Üí current element in array a
    • bIndex ‚Üí current element in array b

    At every step, we compare the two pointed values and move forward only the pointer that contributed the smaller value.

    This guarantees that the merged array remains sorted at all times. 

    
const mergeSorted = (a, b) => {
  let aIndex = 0, bIndex = 0;
  const res = [];

  while (aIndex < a.length && bIndex < b.length) {
    if (a[aIndex] < b[bIndex]) {
      res.push(a[aIndex]);
      aIndex++;
    } else {
      res.push(b[bIndex]);
      bIndex++;
    }
  }

    Once one array is exhausted, we can safely append the remaining elements from the other array because they are already sorted. 

    
  return res
    .concat(a.slice(aIndex))
    .concat(b.slice(bIndex));
};

    Each element is visited exactly once, making this solution both clean and optimal.

    ‚è± Time & Space Complexity

    • Time: O(n + m) ‚Äî one pass over both arrays
    • Space: O(n + m) ‚Äî for the merged result

    This is the best possible complexity for this problem.
